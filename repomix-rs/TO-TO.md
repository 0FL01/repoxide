#### 3. **Потоковая запись вместо буферизации всего output**

**Влияние**: Снижение RAM на ~50%, небольшое ускорение.

#### 4. **Ленивая токенизация (lazy evaluation)**
Токены считаются для ВСЕХ файлов, включая тех что потом сортируются и отбрасываются. Можно:
- Считать токены только для top-N файлов
- Или использовать приблизительный подсчёт (chars/4) для предварительной сортировки

#### 5. **Memory-mapped files для больших файлов**
```rust

// Для файлов > 1MB использовать mmap вместо fs::read()
```

### 🟢 Низкий приоритет (micro-optimizations):

#### 6. **Переиспользование TokenizerState**
Текущий `tiktoken-rs` кеширует BPE, но можно минимизировать overhead вызовов.

#### 7. **SIMD для подсчёта символов**
Использовать `memchr` или `bytecount` для ускорения обработки UTF-8.

#### 8. **Батчинг файлов при параллельной обработке**
Группировать мелкие файлы в батчи для уменьшения overhead от Rayon при работе с тысячами мелких файлов.
